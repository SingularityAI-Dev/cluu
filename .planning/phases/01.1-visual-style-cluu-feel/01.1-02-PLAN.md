    Create `docs/encounter-object-authoring.md` with the following sections (in this order):

    ### Top matter
    - Title: `# Encounter Object Authoring Spec`
    - Subtitle: `Authoring contract for every encounter-object sprite in Phases 2–4.`
    - Frontmatter-style table: `Phase introduced: 01.1`, `Applies to: every encounter-object sprite ("Describe", "Request", "Contract", "Tool", capstone)`, `Owner: solo dev`, `Review cadence: update alongside any new encounter mechanic`.

    ### Section 1: Canvas conventions (per D-21)
    - Default canvas: **48×48** (matches Cluu silhouette + tile grid on 16-pixel half-offsets).
    - Optional larger canvas: **64×64** for encounter objects that legitimately need more room (e.g., a workbench). Must still align to the 32-pixel tile grid — centre of object lands on a tile centre (16, 16 in local coords).
    - Multi-frame canvas: if the encounter has before/after states as separate frames on a sheet, sheet is `48 × (48 × frame_count)` vertically stacked. Frame order: index 0 = withered / default / broken state; index 1 = revived / resolved / repaired state.
    - Rationale: `spacing.encounter = 48` is exported from `lib/design/tokens.ts` (D-12). Changing the default requires updating that token + this spec + every existing sprite.

    ### Section 2: Palette discipline (per D-22 + Plan 01)
    - RULE: every colour in an encounter sprite must correspond to a named token in `lib/design/tokens.ts`. No off-palette pigments.
    - The Vitest palette audit (`tests/design/palette-audit.test.ts`, Plan 05) enforces this for TS/TSX/CSS. Encounter sprite enforcement is **manual visual QA** against the style guide PNG (per RESEARCH §Pitfall 7 — automated pixel-palette audit is out of scope for Phase 01.1).
    - Cluu stays in the `cluu_body` family regardless of biome. Encounter objects live in the biome's palette (Meadow triad for v1 Sunflower).
    - Example table: `Sunflower petals → palette.meadow_sunflower_petal` (future token, added by Plan 03 if the authored sprite introduces a new hue); `Sunflower stem → palette.meadow_grass_shadow` (existing token); `Wilted leaves → palette.ink_700` (grey-green).

    ### Section 3: Before/After state composition (for Describe encounters)
    - Before state: the "needs-intervention" visual. Sunflower withered = drooping head, grey-brown stem, no petals. Communicates "this is the problem."
    - After state: the "solved" visual. Sunflower revived = upright, yellow petals, healthy green stem, optional celebration detail (e.g., a bee).
    - Silhouette rule: the SAME silhouette bounding box applies to before + after. Same 48×48 cell, same approximate shape — only the internal colour/detail changes. Rationale: Phaser swaps the texture key in-place (`sprite.setTexture('encounter_meadow_sunflower_revived')`); if silhouettes shift, the hit-area drifts.
    - Alignment rule: the base of the object (stem root, NPC feet, workbench bottom edge) pins to the bottom-centre pixel of the 48×48 frame. Consistent alignment means scene-side positioning doesn't need per-encounter offsets.

    ### Section 4: File naming convention (per D-19 literal)
    - Pattern: `encounter_<biome>_<id>_<state>.png`
    - `<biome>`: one of `meadow`, `village`, `workshop`, `tidepools`, `library`
    - `<id>`: lowercase snake_case, matches the `.logic.md` encounter id (Phase 2 content)
    - `<state>`: `withered` / `revived` for Describe; `waiting` / `resolved` for Request; `broken` / `repaired` for Contract; `full` / `empty` for Tool; encounter-specific for capstones
    - Example: `encounter_meadow_sunflower_withered.png`, `encounter_meadow_sunflower_revived.png`
    - PNG files live in `public/sprites/` (served by Next.js); `.ase` sources live in `design/aseprite/encounter_<biome>_<id>.ase` (not served — committed for editability per D-20).

    ### Section 5: Aseprite export settings (per RESEARCH §Pattern 1 + §Pitfall 1)
    Copy the 7-checkbox settings VERBATIM from RESEARCH.md:
    1. `File → Export Sprite Sheet`
    2. **Layout**: Sheet type = **Packed**, Constraints = **None**, check **Merge Duplicates**
    3. **Sprite**: Layers = **Visible layers**, Frames = **All frames** (or tag subset)
    4. **Borders**: check **Trim Sprite** + **Trim Cells**; Border Padding / Spacing / Inner Padding ≥ 1
    5. **Output**: **Output File** (PNG), **JSON Data** (Hash or Array), **Tags** checked in Meta options, **Item Filename** = `{frame}`
    6. Tags-required warning: if the sprite has ≥ 1 animation tag, the "Tags" meta checkbox MUST be on. Otherwise `createFromAseprite` silently creates zero animations (Pitfall 1 signature). Verification: `jq '.meta.frameTags | length' public/sprites/<name>.json` returns ≥ 1.
    7. Verification command after export: `file public/sprites/<name>.png` should report the expected dimensions (48×48 × frame count for stacked vertical; larger for Packed layouts — note: Packed layout may reshuffle, so dimension check is a sanity floor, not a tight equality).

    ### Section 6: Authoring at 1:1 display resolution (per RESEARCH §Pitfall 4)
    - AUTHOR at the exact display size: 48×48 for Cluu + encounter objects, 32×32 for tiles. DO NOT author at 96×96 and downscale. Pixel-art is the opposite discipline from 3D art.
    - Reason: mixing `pixelArt: true` with non-integer runtime scaling produces fuzzy edges that `pixelArt` was supposed to prevent.

    ### Section 7: Commit convention (per D-20)
    - Commit both the PNG (runtime asset) AND the `.ase` source (authoring asset). Never commit only the PNG — future-you editing the sprite needs the layered source.
    - `.ase` files are NOT served to the browser. They live under `design/aseprite/` which is NOT in `public/`. Verification: `find public -name '*.ase'` returns zero results (Pitfall 3).

    ### Section 8: Worked example — Withered Sunflower
    - Canvas: 48×48, single frame per state (two separate PNGs rather than a stacked sheet — Describe encounters are state-swap, not animation).
    - Files:
      - `public/sprites/encounter_meadow_sunflower_withered.png` (48×48)
      - `public/sprites/encounter_meadow_sunflower_revived.png` (48×48)
      - `design/aseprite/encounter_meadow_sunflower.ase` (source with 2 frames, tagged `withered` + `revived` OR exported as two files via `--split-tags`)
    - Phaser integration: `this.load.image('encounter_meadow_sunflower_withered', '/sprites/encounter_meadow_sunflower_withered.png')` and the revived variant. Scene swaps texture on encounter-resolve (Phase 2 wires this — out of scope for Phase 01.1).
    - Alignment: stem root pinned at local coords (24, 47). Revived petals extend upward from local (24, 20) to (24, 47). Same silhouette bounds.

    ### Section 9: Out of scope (deferred references)
    - Audio cues per encounter — separate spec later.
    - Motion/tween feel (petals unfurling, etc.) — Phase 2+ decision per encounter.
    - Automated pixel-palette audit — Phase 5 if it ever ships; Phase 01.1 relies on manual QA + the style-guide PNG (D-13).
