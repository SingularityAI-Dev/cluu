# Phase 01.1: Visual style + Cluu feel — Research

**Researched:** 2026-04-21
**Domain:** Pixel-art sprite production pipeline + Phaser 3.90 animation APIs + TypeScript design-token authoring
**Confidence:** HIGH (most claims verified directly against installed `phaser@3.90.0` `.d.ts` + Phaser/Aseprite official docs; some workflow claims are MEDIUM)

## Summary

The shipped `phaser@3.90.0` in this repo already has every API this phase needs: `this.load.aseprite()` consumes an Aseprite-native spritesheet + JSON (Hash or Array) directly; `this.anims.createFromAseprite(key)` creates animations from Aseprite tags, so a single `.ase` file authored with tags `breath`, `blink`, `head_turn` compiles into three Phaser animations with zero hand-written frame indices. This lets the planner treat Aseprite's own tag timeline as the animation spec — the cleanest possible solo-dev pipeline.

The `pixelArt: true` ↔ "loose pixel-inspired" tension flagged in CONTEXT.md D-07 has an authoritative answer: Phaser 3 documents **three distinct rendering modes** — strict pixel-art (`pixelArt:true, antialias:false`), smooth (`pixelArt:false, antialias:true`), and a purpose-built "loose pixel art" mode (`antialias:true, pixelArt:false`) that preserves blocky pixels while smoothing edges between them. The loose mode is only visually distinguishable when sprites are scaled; at the current 1:1 display (game width 768 on a 768-px canvas at FIT scale) all three render identically on integer-dpr devices. Recommendation: keep `pixelArt: true` for Phase 01.1; revisit only if authored sprites visibly clip at iPhone SE 2020's 2× DPR.

Aseprite exports are committed as `.ase` sources per D-20, and the planner can optionally wire a manual `pnpm sprites:export` script that runs `aseprite -b design/aseprite/*.ase --sheet public/sprites/... --data public/sprites/....json --format json-hash` — but since the phase budgets 4–6 sprites across the whole scope, committing both `.ase` and the pre-exported `.png + .json` and re-running CLI manually on edits is simpler than automating.

**Primary recommendation:** Ship the idle animation as a single `cluu_content.ase` with three Aseprite tags (`breath`, `blink`, `head_turn`); export via Aseprite's Packed sheet + JSON Hash + Tags workflow; load with `this.load.aseprite('cluu_content', '/sprites/cluu_content.png', '/sprites/cluu_content.json')`; play breath as the looping baseline and schedule blink / head_turn via `scene.time.delayedCall()` with jittered intervals. Palette lives in `lib/design/tokens.ts` as a single `as const` object with named keys; a Vitest test + ripgrep guard enforces the "no raw hex outside tokens.ts" rule. Keep `pixelArt: true`, author at 1:1, re-evaluate only if a visual regression surfaces on device.

### Pattern 1: Aseprite-native sprite loading (Phaser 3.90)

**What:** Load a `.png + .json` pair exported from Aseprite; Phaser auto-parses frames + tags; `createFromAseprite` builds all Animations in one call.

**When to use:** Any sprite with 2+ frames or ≥1 animation tag. The idle Cluu is the textbook case.

**Example:**
```typescript
// game/scenes/BootScene.ts
// Source: node_modules/phaser/types/phaser.d.ts:66083 + :1258
preload() {
  this.load.aseprite(
    'cluu_content',
    '/sprites/cluu_content.png',
    '/sprites/cluu_content.json'
  );
  this.load.image('grass', '/sprites/grass_32.png');
  // ... other blank cosmetic slots unchanged
}

create() {
  // Creates ALL tag-based animations globally on the AnimationManager.
  // After this, any Sprite using the 'cluu_content' texture can play
  // 'breath' / 'blink' / 'head_turn' by name.
  this.anims.createFromAseprite('cluu_content');

  this.scene.start('MeadowScene');
  this.scene.launch('UIScene');
  bus.emit('game:ready', { timestampMs: Date.now() });
}
```

**Aseprite export settings (MUST match these exactly):**
[CITED: phaser `.d.ts` lines 1194–1220, verified against aseprite.org/docs/cli]
1. File → Export Sprite Sheet
2. **Layout** tab: Sheet type = **Packed**, Constraints = **None**, check **Merge Duplicates**
3. **Sprite** tab: Layers = **Visible layers**, Frames = **All frames** (or tag subset)
4. **Borders** tab: check **Trim Sprite** + **Trim Cells**; Border Padding / Spacing / Inner Padding ≥ 1
5. **Output** tab: **Output File** (PNG), **JSON Data** (Hash or Array — Phaser accepts both), **Tags** checked in Meta options, **Item Filename** = `{frame}` (nothing else)

**Aseprite tag naming convention for this phase:**
- `breath` — loops infinitely, 2–3 frames, ~1.5s cycle (frame duration set in Aseprite)
- `blink` — plays once on demand, 3 frames
- `head_turn` — plays once on demand, 3–5 frames

Phaser's `createFromAseprite` reads each tag's `direction` (forward/reverse/ping-pong) and `repeat` metadata from the JSON. Setting the tag as a **loop** in Aseprite's timeline is sufficient to make Phaser repeat it.

### Pattern 2: Idle animation state machine via `scene.time.delayedCall`

**What:** `breath` runs as the infinite baseline; `blink` and `head_turn` pre-empt breath on random schedulers.

**Why over alternatives:**
- `anims.chain()` — not ideal, chains play strictly after the current animation ends; can't jitter intervals.
- One long spritesheet with manual indices — fights Aseprite tag workflow; brittle when re-exporting.
- Multiple short anims + scheduler — **chosen**; matches how Aseprite authors think (tags = named anims), trivial to re-export without code changes.

**Example:**
```typescript
// game/entities/Cluu.ts — additions inside constructor
// Source: phaser .d.ts Sprite.anims (line 29227, 33057, 36701) + Scene.time

// After adding the base sprite:
this.base.anims.play('breath'); // loops forever

// Schedule blink every 5–8s with random jitter.
this.scheduleBlink(scene);
this.scheduleHeadTurn(scene);
}

private scheduleBlink(scene: Phaser.Scene): void {
  const delay = Phaser.Math.Between(5000, 8000); // D-16 range
  scene.time.delayedCall(delay, () => {
    // anims.play(key, ignoreIfPlaying) — playing a one-shot over the
    // looping breath temporarily replaces it. After blink completes,
    // listen once and restart breath.
    this.base.once('animationcomplete-blink', () => {
      this.base.anims.play('breath');
    });
    this.base.anims.play('blink');
    this.scheduleBlink(scene); // re-schedule next
  });
}

private scheduleHeadTurn(scene: Phaser.Scene): void {
  const delay = Phaser.Math.Between(20000, 40000); // D-16 range
  scene.time.delayedCall(delay, () => {
    this.base.once('animationcomplete-head_turn', () => {
      this.base.anims.play('breath');
    });
    this.base.anims.play('head_turn');
    this.scheduleHeadTurn(scene);
  });
}
```
[VERIFIED: phaser `.d.ts`] Events `animationcomplete-<key>` are standard per-animation events emitted by `Phaser.GameObjects.Sprite`.

### Pattern 3: Sprite-inside-Container animations (critical integration)

**What:** `Cluu` is a `Phaser.GameObjects.Container` with 5 child `Sprite`s. Q: does animation playback work on a child Sprite?

**Answer:** YES. [VERIFIED: `phaser.d.ts` line 36701 — `Phaser.GameObjects.Sprite.anims: AnimationState`] A Sprite child inside a Container carries its own `AnimationState` identical to a top-level Sprite. Container membership affects only transform (position/rotation/scale inherit from parent) — it does not intercept or proxy animation ticks. Calling `this.base.anims.play('breath')` on the child Sprite `this.base` inside the `Cluu` Container is semantically identical to calling it on a standalone Sprite.

**Implication:** Phase 01.1 does NOT need to restructure the 4-layer Container. The `base` child Sprite simply switches from using a single-image texture (`'cluu_base'`) to a sprite-sheet texture (`'cluu_content'`) + starts an animation loop. All cosmetic-slot children (head/body/back/eyes) remain blank single-image placeholders for Phase 3.

### Pattern 4: Design-token file shape (TypeScript)

**What:** Single `const` object, `as const` for literal narrowing, typed consumers via `typeof`.

**Example:**
```typescript
// lib/design/tokens.ts
// Source: typescriptlang.org/docs/handbook/release-notes + design-tokens.org pattern

/**
 * Canonical design tokens for Cluu.
 *
 * RULE: No raw hex codes anywhere outside this file. The palette-audit test
 * (tests/design/palette-audit.test.ts) grep-fails the build if a hex appears
 * in any other .ts/.tsx/.css file.
 */
export const palette = {
  // --- Cross-biome cores (5) ---
  ui_paper: '#faf8f3',      // ← preserves inline `background: #faf8f3` in app/globals.css
  ink_900: '#2d2a26',       // ← preserves inline `color: #2d2a26`
  meadow_hint: '#e8f4d7',   // ← preserves `backgroundColor: '#e8f4d7'` in game/config.ts + CanvasSkeleton
  accent_green: '#2d6a4f',  // ← preserves anchor + primary-button `#2d6a4f`
  cluu_body: '#f4d1b5',     // D-10 warm cream/peach

  // --- Meadow palette (D-09 triad) ---
  meadow_grass_mid: '#6fa872',
  meadow_grass_highlight: '#9bc49e',
  meadow_grass_shadow: '#42744b',
  // planner extends with meadow flower / dirt / shadow hues as needed
} as const;

export type PaletteToken = keyof typeof palette;
export type PaletteHex = (typeof palette)[PaletteToken];

/**
 * Phaser config (e.g. `backgroundColor` in game/config.ts) wants a number, not a hex string.
 * Convert via this helper so scene code stays token-referential.
 */
export function hexToInt(hex: `#${string}`): number {
  return parseInt(hex.slice(1), 16);
}

// Spacing (D-12)
export const spacing = {
  tile: 32,   // D-10 locked tile size
  cluu: 48,   // D-13 locked Cluu size
  encounter: 48, // D-21 encounter-object canvas default (planner confirms)
} as const;

// Radii (D-12 — forward-looking for Phase 5 UI polish)
export const radii = {
  sm: 4,
  md: 8,
  lg: 16,
} as const;
```

**Why `as const` + object (not enum):**
- TypeScript enums are a runtime JS construct; `as const` literal objects are cleaner, tree-shake to nothing when unused. [CITED: typescriptlang.org/docs/handbook/2/enums.html "Prefer literal unions to enums"]
- `typeof palette` extraction gives free discriminated-union autocomplete for consumers.
- `satisfies` is unnecessary here — no external schema to validate against — but could be layered if the Design Tokens Format Module (designtokens.org) shape is adopted later.

**Consumer examples:**
```typescript
// game/config.ts
import { palette, hexToInt } from '@/lib/design/tokens';
backgroundColor: hexToInt(palette.meadow_hint),

// game/scenes/MeadowScene.ts
this.cameras.main.setBackgroundColor(hexToInt(palette.meadow_hint));

// app/globals.css — CANNOT import TS; the audit test treats globals.css as an allowed exception
//   OR you mirror palette values into CSS variables in a separate step.
// RECOMMENDED: generate a CSS file from tokens.ts at build time via a tiny `scripts/tokens-to-css.ts`
// OR: accept D-05 defers React overlay restyle, so globals.css stays hex-literal and is
//     explicitly allowlisted by the audit test.
```

**Planner decision point:** CSS files cannot `import` TypeScript. Two options:
1. **Allowlist** `app/globals.css` (and any .css file) in the audit test with a comment explaining why (defer until Phase 5).
2. **Generate** `lib/design/tokens.css` from `tokens.ts` via a Node script run in `prebuild`. Nicer; adds a build step.

Recommendation: **option 1 for Phase 01.1** (respects D-05 defer). Option 2 during Phase 5 overlay restyle.

### Anti-Patterns to Avoid
- **`new Phaser.GameObjects.Sprite()` — don't construct by hand.** Use `scene.add.sprite()` — the existing `Cluu.ts:33-37` is correct. New code must match.
- **Hard-coding frame indices in `generateFrameNumbers({ start: 0, end: 2 })`** — fragile when re-exporting Aseprite. Use tag names via `createFromAseprite` instead.
- **Calling `anims.play` on the Container, not a child.** `Phaser.GameObjects.Container` does not have an `anims` property — only Sprites do.
- **Mixing `this.load.spritesheet()` and `this.load.aseprite()` for the same asset.** Pick one per sprite. Aseprite loader is richer (parses tags).
- **Authoring at 24×24 and scaling to 48×48 at runtime.** Violates D-07 "author at display resolution" guidance; introduces blur on any non-integer device pixel ratio.
- **Naming palette tokens by colour, not role.** `palette.green700` couples consumers to pigment; `palette.meadow_grass_shadow` binds them to intent. Use role-named tokens (D-11 literal).

## Common Pitfalls

### Pitfall 1: Aseprite "Tags" checkbox missed in Output → Meta options
**What goes wrong:** Exported JSON has `frames[]` but no `meta.frameTags[]`. `createFromAseprite` silently creates zero animations.
**Why it happens:** Default Aseprite export does not include Tags meta. Easy to miss.
**How to avoid:** Document the 7 checkbox settings in `docs/encounter-object-authoring.md` (D-21). Acceptance criterion: `jq '.meta.frameTags | length' public/sprites/cluu_content.json` returns ≥ 1.
**Warning signs:** In-game Cluu renders but is static. Console shows no error. `this.anims.exists('breath')` returns `false`.

### Pitfall 2: `pixelArt: true` + non-integer scale mode
**What goes wrong:** On iPhone SE 2020 (375×667 CSS px, 2× DPR), `Phaser.Scale.FIT` with a 768-wide game scales to 375 CSS px (= 750 device px), a non-integer ratio of 0.977×. With `pixelArt: true`, nearest-neighbor rendering produces visible pixel shimmer on sprite edges during camera follow.
**Why it happens:** FIT chooses the closest size that preserves aspect ratio — it does not snap to integer multiples.
**How to avoid:** Test on a real iPhone SE 2020 (or Chrome DevTools device emulation at 375×667 @ 2×). If shimmer appears: (a) switch scale mode to `Phaser.Scale.ENVELOP` (crops instead of scales) — probably too aggressive for this game, OR (b) flip to "loose pixel art" mode (`pixelArt: false, antialias: true`) — sprites smooth at sub-pixel boundaries. [CITED: docs.phaser.io/api-documentation/class/core-config, phaser.discourse.group/t/roundpixels-is-causing-jittering]
**Warning signs:** Edges of Cluu shimmer/crawl as the camera lerps.

### Pitfall 3: Shipping `.ase` as served asset by accident
**What goes wrong:** Aseprite source file ends up in `public/`, browser tries to load it as an image, 500KB download per visitor.
**Why it happens:** Copy-paste fumble during export. `.ase` files are NOT in `.gitignore` (D-20 commits them).
**How to avoid:** Acceptance criterion: `find public -name '*.ase'` returns zero results. Sources live ONLY in `design/aseprite/`.
**Warning signs:** Large PNG-sized files under `public/sprites/` with `.ase` extension.

### Pitfall 4: Loose-pixel authoring at non-1:1
**What goes wrong:** User authors the Cluu at 96×96 (2× the display size) assuming it will downscale cleanly. Phaser renders at 48×48 and the Canvas context's default `imageSmoothingEnabled` interacts with `pixelArt: true` in confusing ways.
**Why it happens:** "Author large, downscale later" is a common 3D-art habit. Pixel-art is the opposite discipline.
**How to avoid:** Author Cluu at **exactly 48×48** and grass at **exactly 32×32**. Document in D-21 spec. Acceptance criterion: `identify public/sprites/cluu_content.png` reports 48×N (where N = 48 × frame_count).
**Warning signs:** Fuzzy edges on Cluu that do NOT go away by toggling `pixelArt`.

### Pitfall 5: Raw hex leak into Phaser scene code
**What goes wrong:** Developer writes `this.cameras.main.setBackgroundColor(0xe8f4d7)` instead of `setBackgroundColor(hexToInt(palette.meadow_hint))`. Palette later updates; background doesn't.
**Why it happens:** Phaser's API accepts numbers directly — very easy to drop a literal in.
**How to avoid:** The D-22 colour-audit grep MUST cover `0x[0-9a-fA-F]{3,6}` **in addition to** `#[0-9a-fA-F]{3,6}`. Game code uses the `0x` form; CSS uses `#`.
**Warning signs:** Audit greps miss integer literals.

### Pitfall 6: 4-layer Container blank slots referenced before textures load
**What goes wrong:** `Cluu.ts` constructs all 5 sprites in the ctor; if any texture key isn't in the Texture Manager yet, Phaser renders the "__MISSING" red square.
**Why it happens:** MeadowScene instantiates `new Cluu(...)` BEFORE switching from BootScene (existing Phase 1 code is correct — Boot fully preloads + waits before `this.scene.start('MeadowScene')`).
**How to avoid:** Do NOT refactor the BootScene → MeadowScene handoff. Keep all new sprite keys in `BootScene.preload()`. If renaming `cluu_base` → `cluu_content`, update BOTH the preload load call AND every `'cluu_base'` string literal in `Cluu.ts`.
**Warning signs:** Red square where Cluu should be. Console: "Texture key 'cluu_content' not found."

### Pitfall 7: Size check passes, but PNG is indexed colour with stale palette
**What goes wrong:** Aseprite exports in indexed-color mode with its current palette. If the palette in the `.ase` file doesn't match the locked tokens, the sprite looks "off" but dimension tests pass.
**Why it happens:** Manual cleanup in Aseprite trusts the user's eye, not an automated check.
**How to avoid:** Optional tertiary audit — a Node script that runs `sharp(file).raw()` and verifies every pixel's RGB exists in the palette. High complexity for a phase this small; recommend manual visual QA against the style guide PNG instead. Document in D-21.
**Warning signs:** Style guide and game don't match when viewed side by side.

### Example 3: Vitest palette audit (D-22 implementation)
```typescript
// tests/design/palette-audit.test.ts
// Verifies no raw hex or 0x-int colour literals exist outside lib/design/tokens.ts.
// Source: Vitest 4 API (already pinned in package.json)
import { describe, it, expect } from 'vitest';
import { readdirSync, readFileSync, statSync } from 'node:fs';
import { join, relative } from 'node:path';

const ROOT = process.cwd();
const ALLOWLIST = new Set([
  'lib/design/tokens.ts',
  // Phase 01.1 defers React overlay restyle per D-05 — globals.css and the four
  // inline-styled UI components keep their literals until Phase 5 polish.
  'app/globals.css',
  'app/global-error.tsx',
  'app/page.tsx',
  'app/auth/signin/page.tsx',
  'app/play/CanvasSkeleton.tsx',
  'ui/ConsentBanner.tsx',
  'ui/SettingsMenu.tsx',
]);
const EXTS = new Set(['.ts', '.tsx', '.css']);
const HEX = /#[0-9a-fA-F]{3,8}\b|\b0x[0-9a-fA-F]{3,8}\b/;

function walk(dir: string, out: string[] = []): string[] {
  for (const name of readdirSync(dir)) {
    if (name === 'node_modules' || name === '.next' || name === 'coverage' || name.startsWith('.')) continue;
    const path = join(dir, name);
    const stat = statSync(path);
    if (stat.isDirectory()) walk(path, out);
    else if (EXTS.has(path.slice(path.lastIndexOf('.')))) out.push(path);
  }
  return out;
}

describe('palette audit (D-22)', () => {
  it('no raw hex or 0x colour literals outside tokens.ts', () => {
    const offenders: string[] = [];
    for (const file of walk(ROOT)) {
      const rel = relative(ROOT, file);
      if (ALLOWLIST.has(rel)) continue;
      const content = readFileSync(file, 'utf8');
      content.split('\n').forEach((line, i) => {
        // Ignore `//` comments only when the hex is the only hex on the line (pragmatic).
        if (HEX.test(line) && !line.trim().startsWith('//')) {
          offenders.push(`${rel}:${i + 1}  ${line.trim()}`);
        }
      });
    }
    expect(offenders, `Found raw colour literals:\n${offenders.join('\n')}`).toHaveLength(0);
  });
});
```

| Old approach | Current (2026) | When changed | Impact |
|--------------|----------------|--------------|--------|
| Sprite animations via `generateFrameNumbers({start, end})` hand-written in preload | `this.load.aseprite()` + `this.anims.createFromAseprite()` pulling tag metadata | Phaser 3.60+ (stable since 2023) | Removes brittle frame-index math. |

- Phaser's `this.load.atlas()` with TexturePacker JSON — still supported, but Aseprite loader is direct-to-authoring-tool for pixel-art teams.

- `Phaser.GameObjects.Sprite.play(key)` — still works, but prefer `sprite.anims.play(key)` for explicit AnimationState access (matches current `.d.ts` conventions).
