<decisions>
## Implementation Decisions

### Scope & deliverables
- **D-01:** Phase ships THREE artifacts: (a) the §16 single-image style guide, (b) `lib/design/tokens.ts` with named palette + spacing + radii, (c) replaced production sprites in `public/sprites/` for the Meadow biome and Cluu base. The current ~88-byte placeholder PNGs are deleted in favour of real art.
- **D-02:** Phase depth is a **medium pass**: 4–7 days of calendar time, 4–6 plans. Treat as a scope-contained insertion that unblocks Phase 2, not a full art-direction exploration sprint.
- **D-03:** Cluu sprite production in this phase is **Content pose only**. Other moods (Stoked/Content/Sleepy/Blue) and encounter states (Curious/Sparkling) are Phase 3 (CLUU-02, CLUU-03) — producing mood art before mood wiring is premature. The art style established here must demonstrably flex to the other moods later, but we only prove "Content" now.
- **D-04:** Phase 2 encounter-object art for the Withered Sunflower (withered state + revived state) IS produced in this phase alongside a reusable encounter-object authoring spec. Rationale: it's the single load-bearing encounter of Phase 2; shipping it with the style guide means Phase 2's planner focuses on the grading pipeline, not art direction derivation.
- **D-05:** React overlays (`app/page.tsx` landing, `app/auth/signin/page.tsx`, `ui/ConsentBanner.tsx`, `ui/SettingsMenu.tsx`) are **NOT restyled in this phase**. They stay on the current `#faf8f3` / `#2d2a26` / `#2d6a4f` inline-styled minimalism. This is a deliberate deferral to keep scope tight — tracked in deferred.

### Art direction
- **D-06:** Visual mood is **chunky pixel art** — indie-cozy, crunchy 32×32 tiles, 48×48 Cluu. Reference feel: Stardew Valley / Eastward / Celeste. Not strict pixel-purism (see D-07).
- **D-07:** **Loose pixel-inspired discipline**: diagonal anti-aliasing is allowed, half-pixel shading is allowed, freeform outlines are allowed. This is an intentional choice — easier to author solo, avoids the strict-pixel-art "content expectation" trap for a game with 27–37 encounters. **Implication to verify:** `game/config.ts` currently sets `pixelArt: true` (forces nearest-neighbor + integer scale). The planner MUST evaluate whether to keep `pixelArt: true` (if loose-pixel sprites are authored at the exact display resolution so nearest-neighbor is neutral) or flip it to `pixelArt: false` / switch to `antialias: true` (to allow sub-pixel AA to render cleanly). Default recommendation: keep `pixelArt: true` and author sprites at 1:1 display resolution; if that fights the loose aesthetic, revisit as a deviation with explicit test on iPhone SE 2020.
- **D-08:** Color palette system is **biome-palette**: 5 cross-biome cores (Cluu body, UI text, UI background, ink, accent) + ~4–8 colours per biome × 5 biomes. Overall target: **~32–48 named colours total** across the whole game. This phase ships the 5 cores + the Meadow biome subset only.
- **D-09:** Meadow green triad is locked: **`meadow_grass_mid: #6fa872`, `meadow_grass_highlight: #9bc49e`, `meadow_grass_shadow: #42744b`**. Saturated warm cozy, reads as "afternoon grass", pairs with the existing `#faf8f3` cream background.
- **D-10:** Cluu's base body colour is a **warm cream/peach**, around `#f4d1b5`. Rationale: reads distinct against every biome palette (green Meadow, brown Village, wood Workshop, blue Tide Pools, amber Library), keeps Cluu as the focal point across the game, works with cosmetic pattern overlays (accent colour does the visual work; base stays neutral). Planner may adjust the exact hex to harmonise with the final Meadow palette — but the family (warm cream/peach) is locked.

### Palette tokens & documentation
- **D-11:** Palette lives in `lib/design/tokens.ts` as named exports with semantic + swatch names. Shape: `export const palette = { cluu_body: '#f4d1b5', meadow_grass_mid: '#6fa872', ink_900: '#2d2a26', ui_paper: '#faf8f3', ... } as const;` with a typed `Palette = typeof palette`. Every downstream sprite / UI / style guide asset references a token name, not a raw hex.
- **D-12:** Tokens file also exports semantic spacing (e.g., `spacing.tile = 32`, `spacing.cluu = 48`) and radii (for future UI) so later React overlay work has the same single source of truth — even though D-05 defers the actual overlay restyle.
- **D-13:** Style-guide artifact is a **single PNG image** at `design/style-guide.png` (or `docs/visual-style-guide.png` — planner picks, not critical). Contains: named colour palette strip with swatch + hex + token name, Cluu reference sheet (Content pose + idle-animation frames breakdown), Meadow tile samples (all variants), Withered Sunflower before/after, encounter-object authoring template. This is the §16 deliverable.

### Cluu's form & pose
- **D-14:** Cluu's silhouette is a **chubby 4-legged creature** — think capybara-ish or a tiny fluffy dog. Rounded body, 4 short legs, defined head, tail optional. NOT a floating sprite, NOT a slime blob. Fills most of 48×48 with clear leg / body / head regions so cosmetic slots (head, body pattern, back) are structurally separable. Legs are part of the base sprite, not a cosmetic slot.
- **D-15:** Content pose visual language: **head slightly tilted + bright neutral eyes** (not squinted, not wide, no mouth — or a very subtle mouth if it reads clean at this resolution). Tilt reads "attentive, listening". Phase 3's Curious state later pushes the tilt further + adds a visual cue (sparkle / `?`), keeping the visual language consistent with Content as the neutral baseline.
- **D-16:** Cluu **idle animation** in Phase 01.1: **breath loop + occasional blink + occasional head turn**. Breath = 2–3 frame vertical bob, ~1.5s cycle. Blink = random every ~5–8s. Head turn = rare (every 20–40s), 2–3 frames, looks around once then returns to Content pose. This is deliberately more animation than the minimum — the phase is literally named "Cluu feel", motion is core to that. Bundle impact is negligible (48×48 PNGs, even 10 frames = ~20 KB).
- **D-17:** Animation authoring is **sprite-frame based**, not rigged/tweened. Frames export from Aseprite as a single spritesheet + JSON Hash (design §8 / §13). Phaser's `anims.create` consumes the spritesheet. No skeletal animation in v1.

### Production pipeline
- **D-18:** Art production method is **AI-generated candidates + manual cleanup in Aseprite**. User prompts an image model (Nano Banana / Sora / Midjourney, user's choice) for pixel-art candidates, picks the best fit, cleans up in Aseprite to hit the locked palette and pixel discipline. Rationale: fast for solo dev, leverages the user's prompting fluency (dogfood signal for the game's own core value), keeps identity control through the cleanup pass. Plans DO NOT call image APIs — this is a human-in-the-loop task.
- **D-19:** Execution split: **plans describe WHAT art assets are needed, user produces the PNGs, plans verify by file-exists + dimensions + sprite-load-success**. Plans do not attempt to auto-generate sprites. Each art task's acceptance criteria include: exact file path, pixel dimensions, Aseprite source file committed alongside the PNG (for future edits), palette-token usage verified by an automated colour-audit script OR manual inspection.
- **D-20:** Aseprite source files (`.ase`) are committed to the repo at `design/aseprite/*.ase` alongside the exported PNGs in `public/sprites/`. Rationale: future-you returning to edit a sprite needs the layered source, not just the flattened PNG. Bundle impact: none (the `.ase` files are not served to the browser, only the exported PNGs hit `public/`).
- **D-21:** Encounter-object authoring spec lives at `docs/encounter-object-authoring.md` (or similar — planner picks exact path). Defines: canvas size conventions (e.g., encounter objects fit within 48×48 or 64×64 tiles aligned to the tile grid), palette-token usage rules, before/after state composition (for Describe encounters like the Sunflower), naming convention (`encounter_<biome>_<id>_<state>.png`), Aseprite template file. Every encounter-object sprite in Phases 2–4 follows this spec.

### Testing & verification
- **D-22:** Visual regression is verified **manually + via a simple colour-audit script**. Automated: a Vitest test or Node script asserts that every palette colour referenced in CSS/inline-styles/sprite metadata appears in `lib/design/tokens.ts`, and that no raw hex codes exist outside `tokens.ts` (grep-based). Manual: user visually confirms the style guide image matches intent + the live `/play` scene renders the production Cluu + production Meadow tiles.
- **D-23:** Phase 01.1 exit gate requires: (a) `design/style-guide.png` exists and visually reviewed, (b) `lib/design/tokens.ts` compiles and is imported by at least one consumer (meadow scene or Cluu entity), (c) `public/sprites/grass_*.png` + `public/sprites/cluu_base.png` + Cluu idle-animation frames exist, (d) Withered Sunflower before/after PNGs exist, (e) encounter-object authoring spec doc exists, (f) `npm run dev` renders the production art at `/play` with no regressions to Phase 1's scaffold behaviour (anchor moves, Cluu follows, magic-link still works).

### Claude's Discretion
- Exact hex values within each biome palette family beyond the Meadow green triad (D-09) and Cluu body family (D-10) — planner/user iterate during art production.
- Exact file names within `public/sprites/` beyond the `grass_*` / `cluu_*` / `encounter_meadow_sunflower_*` conventions implied by D-19.
- Whether the style guide is PNG or multi-page PDF or markdown+embedded images — D-13 locks PNG but planner may propose a better format with rationale.
- Aseprite layer / slice conventions inside the `.ase` files.
- Spacing/radii token values in `lib/design/tokens.ts` beyond `tile=32` / `cluu=48`.
- Exact blink/head-turn animation timing within the ranges in D-16.
- Whether colour-audit is a Vitest test, a standalone Node script, or a grep check in CI (D-22).

### Established Patterns
- **Sprite loading:** every asset loads in `BootScene.preload()` with `this.load.image(key, '/sprites/...')`. Spritesheets for animation load with `this.load.spritesheet(key, '/sprites/...', { frameWidth, frameHeight })` — new Cluu idle animation frames follow this pattern.
- **Phaser Container cosmetic layering:** Cluu stacks 5 sprites in z-order (back → base → body_pattern → head → eyes). New art respects this; each slot sprite must be 48×48 to align without offsets.
- **No CSS framework:** inline-style + globals.css. `lib/design/tokens.ts` is a plain TS constant object, not Tailwind config, not CSS-in-JS variables. Consumers import and spread into inline styles.
- **Scene structure:** Boot → Meadow → UIScene launch pattern (D-07 Phase 1). New biomes in Phase 3 copy this pattern. Phase 01.1 doesn't touch scene structure.

### Integration Points

<specifics>
## Specific Ideas

- Art mood-board references (for the AI-prompting pass, NOT copy-cloning): Stardew Valley sprites for tile feel, Eastward for Cluu's chubby-creature silhouette, Celeste for palette saturation levels, A Short Hike as a pastel-companion contrast reference. The game is its own thing — these are directional hints only.
- The existing `#faf8f3` cream background + `#2d2a26` ink pair is genuinely good and should survive the palette expansion. Treat them as preserved ancestors in `tokens.ts`.
- Cluu's personality from the design doc: curious, never sad, relational-not-punitive. Head tilt + bright eyes should read as "interested in what you're saying", never as "sad you left". The blink and head-turn idle animations reinforce this — Cluu is looking around, alive, not staring blankly.
- The chubby-4-legged form means the `back` cosmetic slot (cape / wings / backpack in Phase 3) needs a clear "back" region on the silhouette. Author the base sprite with the back area slightly visible behind the body so a cape sprite can peek out, not be entirely hidden.

</specifics>
