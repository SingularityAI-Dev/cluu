## Decisions Made

- **48×48 as the canvas default, 64×64 as an opt-in** — rationale: `spacing.encounter = 48` in the tokens file is load-bearing; widening the default requires a lockstep update of token + spec + existing sprites. Made 48×48 the path of least resistance.
- **`--split-tags` export for Describe encounters** — documented as the recommended CLI invocation because it produces `encounter_meadow_sunflower_withered.png` and `encounter_meadow_sunflower_revived.png` directly from a single `.ase` with tagged frames, matching the file-naming convention one-to-one. Avoids manual rename steps.
- **Silhouette-stability rule framed as non-negotiable** — phrased as "hit-area drifts, physics body wrong, Cluu pathfinding anchor misaligns" so future authors see the runtime consequence, not just the aesthetic ask.
- **No allowlist entry for this doc in the palette audit** — decided to keep the spec strictly token-referenced (no `#f4d1b5` examples, no `0x...` integer examples) so Plan 05's audit treats it like any other file. Simpler than allowlist-ing.
- **State vocabulary locked per encounter type** — withered/revived, waiting/resolved, broken/repaired, full/empty. Prevents Phase 2+ authors re-inventing state names and diverging from the `<state>` slot in the filename pattern.
- **Bottom-centre anchor at local (24, 47) across all 48×48 encounter sprites** — documented as a concrete coordinate (not just "align the base"), so scene-side positioning code in Phase 2 can hardcode the offset and have it work for every encounter.
